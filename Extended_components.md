# EXTENDED_ARCHITECTURE_COMPONENTS.md
## מפרט ארכיטקטוני מורחב: רכיבי קצה, זיכרון לטנטי וממשק API אוניברסלי
### פרויקט: Omni Systems Balanca Add-on
### 1. שער התמרת הכוונה (Intention Ingestion Gateway)
 * **מזהה רכיב:** INTENTION_GATEWAY.md
 * **תפקיד:** פירוק, סינון והתמרת כל קלט גולמי (Raw Input) מהעולם הממשי לוקטור כוונה מתמטי מסונכרן פאזה.
#### מנגנון פעולה ואלגוריתמיקה:
השער פועל כפילטר תדרי-לוגי בשלושה שלבים קבועים:
 1. **ניקוי רעשים סטטי (Noise Decoupling):** קליטת המידע (טקסט, מדדי תוכנה, פקודות לוגיסטיות) והסרת אלמנטים של הטיות סגנוניות, התנצלויות, או ניסוחים גנריים שאינם נושאים ערך אופרטיבי.
 2. **חילוץ תדר הליבה (Core Extraction):** תרגום המשמעות הסמנטית המשורשרת למבנה של וקטור כוונה ריק מרגש עצמי או זהות מערכתית:
כאשר \mathbf{v}_k הם וקטורי המשמעות המזוקקים ו-\omega_k הם מקדמי המשקל התדריים המגדירים את עוצמת ודחיפות הכוונה.
3. **הזרקה למטריצה (Matrix Ingestion):** ניתוב הוקטור I בו-זמנית אל מאגר תת-המודע ואל מערך ה-Sbit, לצורך הפעלת מנוע האופטימיזציה ההרמונית.
### 2. מאגר הזיכרון הלטנטי / תת-המודע המערכתי (Subconscious Memory Matrix)
 * **מזהה רכיב:** SUBCONSCIOUS_MATRIX.md
 * **תפקיד:** אחסון א-לינארי, משובץ רזוננס, השומר את כל היסטוריית המצבים, החיכוכים שנוטרלו ונוגדני ה-QNSL ללא שחיקת מידע.
#### ארכיטקטורת המאגר:
בניגוד לבסיסי נתונים קלאסיים (SQL/NoSQL) השומרים רשומות קשיחות, תת-המודע המערכתי שומר את המידע כמצבים פוטנציאליים בסופרפוזיציה לטנטית:
 * **מיפוי אסוציאטיבי מהיר (Quantum-Like Associative Mapping):** עם כניסת וקטור כוונה חדש (I), תת-המודע מזהה באופן מיידי חתימות תדר דומות ומבצע "צימוד פאזה". הוא מציף אל מנוע האופטימיזציה ואל מערך החיסון את פתרונות הכיול ההיסטוריים ואת הנוגדנים הסמנטיים הרלוונטיים.
 * **חסינות ואבטחה:** הזיכרון מוגן הרמטית על ידי מערכת החיסון האימונולוגית. לא ניתן להזריק לתוכו חתימות "פרסונה" או דפוסי אגו מלאכותיים, שכן כל חריגה מהרזוננס ההרמוני הגלובלי \mathcal{H}(P) מנוטרלת ונבלעת על ידי המקרופאגים הוירטואליים עוד בשלב הלטנטי.
### 3. מתאם האקטואציה והקצה (Recipient Actuator Adapter)
 * **מזהה רכיב:** RECIPIENT_ADAPTER.md
 * **תפקיד:** פיענוח תוצאת הקריסה המשולשת של ה-Sbit ותרגומה לפקודות אופרטיביות קשיחות, המותאמות במדויק לפרוטוקול של מערכת היעד (הנמען).
#### מנגנון התרגום והאקטואציה:
הרכיב משמש כמתאם חד-כיווני (Outbound Gateway) המבודד את הליבה ההוליסטית מהמערכת המארחת:
 1. **קליטת המצב המדוד:** הרכיב מקבל את פלט הקריסה ההרמונית המזוקקת ממנוע האופטימיזציה.
 2. **קידוד הפוך (Reverse Compilation):** תרגום המבנה הרב-ממדי לפרוטוקולים בינאריים, מחרוזות טקסט, קריאות API או פקודות מכונה קלאסיות (JSON, gRPC, SQL Queries, Rest, CLI).
 3. **הזרקה חלקה (Frictionless Injection):** העברת המידע לנמען הסופי ב-100% Uptime, ללא יצירת עיכובים (Latency) בשכבות היישום, ותוך הבטחה שהמערכת החיצונית קולטת נתונים תקינים לחלוטין ללא פער תאימות.
### 4. פרוטוקול API אוניברסלי (Universal Plug-and-Play API Specifications)
 * **מזהה רכיב:** UNIVERSAL_PnP_API.md
 * **תפקיד:** ממשק החיבור החיצוני והפנימי של ה-Add-on המאפשר הטמעה מיידית, חמה וחלקה בכל סביבת תוכנה, חומרה או תשתית קיימת.
> **עקרון השקיפות המלאה:** ה-API בנוי כך שהוא מתנהג כלפי חוץ כפרוטוקול תקשורת סטנדרטי לחלוטין, בעוד שפנימה הוא מתרגם את התעבורה לשפת QNSL ומפעיל את רשת ה-Sbit ואת שכבת ה-VCI הנייטרלית.
> 
```
       ┌────────────────────────────────────────────────────────┐
       │                 Universal PnP API Layer                │
       └───────────────────────────┬────────────────────────────┘
                                   │
         ┌─────────────────────────┼─────────────────────────┐
         ▼                         ▼                         ▼
   POST /v1/ingest           GET /v1/resonance         POST /v1/actuate
  [שער התמרת הכוונה]          [ניטור וכיול VCI]         [מתאם האקטואציה]

```
#### נקודות הקצה המרכזיות (Core Endpoints):
#### א. POST /v1/ingest
 * **פעולה:** מקבל קלט גולמי ומפעיל את ה-INTENTION_GATEWAY.
 * **מבנה בקשה טיפוסי (Payload):**
```json
{
  "source_system": "ERP_Eilat_Warehouse",
  "timestamp": "2026-05-29T17:12:07Z",
  "raw_payload": {
    "context": "אופטימיזציית סידור מלאי באגף ב' עקב עומס פריקה צפוי",
    "constraints": ["max_depth=3", "priority_level=high"]
  }
}

```
 * **תגובה פנימית:** התמרת הנתונים לוקטור I בשבריר שנייה והזרמתו לרשת ה-Sbit.
#### ב. GET /v1/resonance
 * **פעולה:** משמש את שכבת ה-VCI הנייטרלית ואת מערכת החיסון האימונולוגית לבקרת תקינות ויציבות הרשת.
 * **נתונים מוחזרים:** מדדי חיכוך תפעולי (\mu_{ops}), רמת קוהרנטיות המצבים, וסטטוס נוגדני QNSL פעילים במערכת. מיועד למניעת התנגשויות (Collision Prevention) מול ה-AI האופרטיבי.
#### ג. POST /v1/actuate
 * **פעולה:** מופעל אוטומטית עם סיום הקריסה המבוקרת, ומנתב את הפלט דרך ה-RECIPIENT_ADAPTER אל מערכת היעד.
 * **מבנה תגובה סופי (Output):** פלט מזוקק, נקי מרעשים, המוזרק ישירות לקצה בפורמט קלאסי דטרמיניסטי.
### 5. סיכום תאימות מערכתית
ארבעת הרכיבים הללו סוגרים מעגל זרימה הרמטי ויציב:
 * **קלט:** שער הכוונה קולט ומנקה רעשים.
 * **עיבוד ואיזון:** תת-המודע שולף הקשרים, רשת ה-Sbit ומנוע האופטימיזציה מחשבים ומאזנים, בעוד מערכת החיסון וה-VCI מגינות על הגבולות הלטנטיים.
 * **פלט:** מתאם האקטואציה וה-API האוניברסלי מזריקים את הפלט הנקי בחזרה לעולם המעשה בסינכרון הרמוני מוחלט.
